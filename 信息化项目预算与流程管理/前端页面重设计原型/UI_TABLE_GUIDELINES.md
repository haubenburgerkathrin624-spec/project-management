# 表格与附件列表 UI 规范

本原型里的表格、附件清单、审批材料清单都按同一套规则处理，避免出现表头和内容错位、状态列被挤压、操作按钮掉行等问题。

## 必须遵守

- 表头行和数据行必须复用同一套 `grid-template-columns`，优先用 CSS 变量定义列模板。
- 增加“序号”列时，表头和每一行必须同时增加同一列，不能只给数据行补序号。
- 窄屏可以横向滚动，但不能压缩到标题、状态、操作列错位。
- 状态列要预留足够宽度，不能把“待确认，可补充”等状态挤成难读的碎行。
- 操作列保持为一组按钮，按钮不换行；如果空间不够，由表格容器横向滚动承载。
- 所有单元格使用 `align-items: center`，并设置 `min-width: 0`，长文件名按统一策略截断或换行。
- 必填附件必须用红色 `*` 标识，不能把星号当普通正文。
- “可多附件”“可多选”等技术提示不显示给用户；上传能力保留在交互逻辑里。
- 新增附件行必须复用已有附件表格列模板，不能单独写一套临时布局。

## 检查清单

- 表头和首行数据的单元格数量一致。
- `getComputedStyle(row).gridTemplateColumns` 在同一张表内保持一致。
- 移动端视口下，状态列和操作列不互相覆盖。
- 横向滚动出现时，表头、内容、按钮仍在同一列轨道上。
- 附件表格新增行后，序号、附件名称、要求、状态、操作列仍然对齐。
