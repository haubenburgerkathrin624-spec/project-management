# 信息化项目管理系统改造 PRD（详细版）

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 产品名称 | 信息化项目预算与流程管理系统 |
| 文档类型 | PRD / 产品需求文档 |
| 版本 | V1.0 |
| 编制依据 | 用户手册、两场需求沟通录音、旧系统痛点分析、可行性与详细设计方案 |
| 产品定位 | 面向高校信息化部门的项目预算、流程、资料与归档一体化工作台 |

## 2. 背景与目标

旧系统已具备登录、权限、项目管理、采购管理、实施、试运行、验收、归档、退金、结项、立项审核、验收审核和事务提醒等基础能力。用户反馈显示，系统需要从“按阶段填表和审批”升级为“面向信息化部门治理的轻量化工作台”。

核心目标：

- 少填：复用项目库、模板和历史信息，减少重复录入。
- 少审：领导侧以看板和查询为主，普通节点不强制领导审批。
- 少找：模板、材料清单和项目资料集中管理。
- 好查：年度、预算、状态、负责人、部门、归档状态可快速筛选。
- 好归档：自动汇总材料、识别缺失项、生成归档清单和资料包。

## 3. 产品范围

### 3.1 本期范围

- 首页工作台
- 项目申报
- 项目库
- 年度预算
- 立项与预算联动
- 采购过程资料管理
- 实施过程资料管理
- 试运行管理
- 验收管理
- 归档/移交
- 模板中心
- 领导看板
- 查询统计
- 待办提醒

### 3.2 非本期范围

- 不替代学校采购系统、合同系统、审计系统和档案系统。
- 不实现复杂采购审批分支，只记录必要字段、过程文件和结果文件。
- 不强制做全量历史数据结构化迁移。
- 不在本期建设移动端独立 App。

## 4. 用户角色

| 角色 | 核心诉求 | 典型动作 |
|---|---|---|
| 项目申报人/负责人 | 快速申报、少重复填、知道缺什么材料 | 新建申报、上传资料、查看进度、提交验收 |
| 信息化部门管理员 | 管全局、查预算、催材料、做归档 | 审核入库、立项、预算匹配、材料检查、归档确认 |
| 项目主管/归口负责人 | 对关键节点做专业确认 | 查看负责项目、审核节点、填写意见 |
| 领导 | 看全局、看异常、少点审批 | 查看年度看板、查询重点项目、导出报表 |
| 档案/综合管理员 | 接收完整归档材料 | 查看归档包、确认移交、下载清单 |
| 系统管理员 | 配置用户、角色、菜单、参数 | 维护组织、权限、流程、模板、字典 |

## 5. 核心业务流程

1. 项目负责人全年开放申报，保存草稿并提交。
2. 信息化部门管理员初审，符合要求后进入项目库。
3. 年度预算录入或导入，形成预算池。
4. 管理员在立项页面从项目库选择项目，并绑定预算。
5. 项目进入采购、实施、试运行、验收阶段，各阶段上传材料。
6. 验收通过后进入归档，系统生成缺失材料提醒和归档清单。
7. 材料完整后打包下载并移交档案/综合管理人员。
8. 项目退金/结项，形成完整闭环。

## 6. 状态机

项目状态：草稿、已提交、退回修改、已入库、待立项、已立项、采购中、实施中、试运行中、验收申请中、验收退回、验收通过、归档中、已归档、已结项、已取消。

预算状态：启用、冻结、已占用、部分执行、执行完成、已释放。

材料状态：未上传、已上传、需补正、已确认、已归档。

## 8. 字段设计

关键字段包括项目基础字段、预算字段、材料字段、审核字段和归档字段。字段需支持导入、导出和权限控制。

## 9. 权限矩阵

权限原则：项目负责人看本人，管理员看全量，项目主管看负责范围，领导只读全局，档案人员只看归档。

## 10. 非功能需求

- 单文件上传建议支持 200MB。
- 常用查询 3 秒内返回。
- 操作日志保留关键字段前后变化。
- 附件需支持备份、下载权限控制和版本留痕。
- 系统应兼容主流浏览器。

## 11. 版本规划

MVP：项目申报、项目库、预算、立项联动、阶段材料、归档清单、基础查询。

P1：模板中心、领导看板、缺失提醒、大附件、统计导出。

P2：对接统一身份认证、财务预算、采购平台、档案系统，移动端提醒。

## 12. 验收清单

- 主流程闭环可跑通。
- 预算与立项联动准确。
- 阶段材料能进入归档材料池。
- 归档清单能自动生成。
- 领导无需参与普通审批即可看全局。
- 模板可维护、下载、停用。
- 查询统计可支撑年度汇报。
